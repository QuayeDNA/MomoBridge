How It Works

From SMS to confirmation
in under a second

Customer sends MoMo → your phone intercepts the SMS → app stores it locally. When the customer enters their reference on your website, the widget talks to your phone via a lightweight relay. No cloud database, no third parties.

01

Intercept SMS

App automatically captures MoMo SMS confirmations from MTN, Telecel, AT, and T-CASH. Parses reference, amount, sender.

→
02

Store Locally

Transaction saved in on-device Room database as PENDING. No data ever leaves your phone unless you claim it.

→
03

Customer Claims

Customer enters reference on your website widget. Widget sends claim request to the relay, which forwards it to your phone.

→
04

Instant Result

App verifies against local DB and responds. Confirmed → your site credits the customer. Failed → friendly error shown.

Features

Built for vendors who
need real-time trust

Every feature is designed around one principle: payment verification should be instant, offline-capable, and require zero trust in third-party infrastructure.

⚡

Real-time verification

SMS intercepted, parsed, and ready for verification in under 50ms. Customer gets their result before the toast disappears.

🔑

Multi-key architecture

Generate one API key per store location. All keys route to the same device. Per-key transaction audit trail built in.

📴

Works offline

SMS interception and storage work without internet. Only verification needs the relay. Graceful offline feedback to customers.

🔒

No cloud dependency

Your phone IS the database. The relay is a stateless switchboard with zero storage. No user data on any server, ever.

📱

Multi-network

Works with MTN, Telecel (formerly Vodafone), AT (formerly AirtelTigo), and T-CASH. Config-driven — add new networks without code changes.

🎨

Embeddable widget

Three integration modes: popup modal, inline embed, or redirect page. Drop a <script> tag and you're done.

Architecture

Three components,
zero complexity

No Kubernetes, no microservices, no database migrations. Three pieces of software that fit in a single GitHub repo.

📱 Android App

  • Kotlin + Jetpack Compose + Material 3
  • Intercepts SMS via BroadcastReceiver
  • Parses and stores transactions in Room (SQLite)
  • WebSocket client connects to relay
  • Generates and manages API keys per store
Kotlin Jetpack Compose Room OkHttp

🔁 Relay Server

  • Single Bun process — one file, zero npm deps
  • In-memory WebSocket ↔ apiKey map
  • POST /claim forwards requests to the right device
  • Stateless — no database, no disk writes
  • Deploys to Render free tier or any VPS
Bun TypeScript WebSocket Docker

🌐 Web Widget

  • Vanilla JavaScript — zero dependencies, zero build step
  • Three modes: popup (modal overlay), inline (<div> embed), redirect (standalone page)
  • Exposed as window.MoMoBridge via <script> tag
  • Rich result cards with success/error/warn states
  • postMessage API for parent app integration
  • Served from Vercel CDN with global caching
Vanilla JS HTML/CSS Vercel CDN
Integration

Add to any website
in two lines

Drop a script tag, configure your relay URL and API key, and choose your integration mode. No npm install, no build step, no iframe mess.

1. Load the script

<!-- Add to your checkout page --> <script src="https://momo-bridge.vercel.app/momobridge.js"></script>

2. Listen for result

// Optional: listen for postMessage result window.addEventListener('message', e => { if (e.data.type === 'momo_result') { // { confirmed, reference, amount } if (e.data.confirmed) creditWallet(e.data); } });
Popup

Modal Overlay

Opens a centered verification modal. Best for manual trigger — attach to a "Verify Payment" button.

MoMoBridge.popup({ relayUrl: 'https://your-relay.com', apiKey: 'mb_...', amount: 18.00, onSuccess: handleSuccess, onFailure: handleFailure });
Inline

Embedded Form

Renders the verification form directly into a container div. No modal — stays on the page.

MoMoBridge.inline({ relayUrl: 'https://your-relay.com', apiKey: 'mb_...', amount: 18.00, container: '#verify-container', onSuccess: handleSuccess, onFailure: handleFailure });
Redirect

Standalone Page

Opens the dedicated verification page in a new tab or same window. Supports callback URL.

MoMoBridge.redirect({ relayUrl: 'https://your-relay.com', apiKey: 'mb_...', amount: 18.00, callbackUrl: 'https://myshop.com/checkout', target: '_blank' });

🔁 Deploy the Relay Server

The relay is a single-file Bun server. Deploy it anywhere — Render, Fly.io, Railway, or a $5 VPS.

# One-command deploy (Render) git clone https://github.com/QuayeDNA/MomoBridge cd momobridge/relay bun install bun run index.ts # Environment PORT=8080

Connect the Android app to your relay URL, embed the widget on your site, and payments verify in real time.

# Docker deploy docker build -t momobridge-relay . docker run -p 8080:8080 momobridge-relay # Health check curl https://your-relay.com/health // { "status": "ok", "connections": 1 }
